# Contributing

## Development Flow

1. Crie branch a partir de `main`.
2. Execute validações locais:
   - `npm run doctor`
   - `npm run validate:contracts`
   - `npm run smoke`
3. Abra Pull Request com contexto técnico.

## Pull Request Checklist

- [ ] Escopo está claro e limitado.
- [ ] Contratos de tools atualizados (se aplicável).
- [ ] Documentação atualizada.
- [ ] Sem segredos em código/commits.
- [ ] Passou em checks locais.

## Commit Convention

Use Conventional Commits:

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `chore: ...`
- `refactor: ...`
